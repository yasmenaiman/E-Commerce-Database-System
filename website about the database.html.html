Business Problems Solved

🔄
Loyalty Points Not Reversed on Cancellation
Cancelled orders kept their loyalty points — customers exploited this to collect free points with no purchase.
✓ Trigger: trg_ReverseLoyaltyOnCancel
⚠️
Missing Critical Data Validations
No checks on emails, prices, or promotions — dirty data entered the system freely causing downstream errors.
✓ Strict CHECK Constraints
🐌
Heavy DB from Storing Images Directly
Storing binary images in the database caused severe performance degradation and high storage costs.
✓ URL-based Image Storage
💳
No Tracking for Outstanding Payments
Delivered orders with unpaid balances had no visibility — leading to undetected revenue loss.
✓ Outstanding Payments Query
📦
No Visibility on Cart Abandonment
No way to track how many carts were created vs converted, leaving marketing teams blind to drop-off patterns.
✓ Cart Conversion Rate Query
🔒
Returns Without Delivery Verification
Return requests could be submitted even for undelivered orders, enabling fraudulent return claims.
✓ Trigger: trg_ValidateReturnDelivery

Triggers

trg_ReverseLoyaltyOnCancel
Fires after an order status update. Automatically reverses earned loyalty points when an order is cancelled.
trg_ValidateReturnDelivery
Fires before inserting a return. Blocks the operation if the order has not been marked as Delivered in the shipping system.
// analytics

Views & Queries

V1 vw_CustomerMaster View
Displays customers with total orders and total completed payment amounts.
V2 vw_ProductCatalog View
Shows products with current price, total stock quantity, and stock status (In Stock / Low Stock / Out of Stock).
V3 vw_OrderDetails View
Detailed order info including customer name, product, payment status, and shipment status.
V4 vw_SupportSLA View
Monitors support ticket SLA compliance based on priority (High: 24h / Medium: 48h / Low: 72h).
Q1 Daily Sales Analysis Query
Analyzes daily sales performance including order count, total revenue, average order value, and items sold.
Q2 Cart Conversion Rate Query
Calculates monthly cart-to-order conversion rate to identify abandonment patterns.
Q3 Top 3 Products per Category Query
Uses DENSE_RANK() to identify the top 3 best-selling products in each category based on revenue.
Q4 Payment Method Analysis Query
Analyzes payment methods by transaction volume, total revenue, failed payments, and refund counts.
Q5 Outstanding Payments Query
Identifies delivered orders that have not been fully paid — enabling finance teams to follow up on revenue gaps.
